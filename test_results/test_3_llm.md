## 测试设计

### 测试目标
验证 DeepSeek LLM 能生成比规则模板更具体、更可操作的预警建议，并验证全部降级链路。

### 测试方法
1. **功能测试**: 发送真实 API 请求，验证单条/批量/低风险跳过/端到端
2. **质量评估**: 人眼先写出"理想建议要素"，再对 LLM 输出 4 维度评分（具体性/可操作性/准确性/无幻觉，各 0-3 分，满分 12）
3. **降级测试**: monkeypatch 模拟 7 种故障场景，验证每层回退
4. **稳定性**: 同输入 3 次调用，验证一致性

### 测试数据

| 数据 | 类型 | 地区 | 产品 | 评分 | 用途 |
|------|------|------|------|------|------|
| 小麦赤霉病预警 | 病虫害 | 河南 | 小麦 | 78 | A1, B1, D1 |
| 强降雨预警 | 气象灾害 | 河南 | 未识别 | 72 | A2, B2 |
| 蔬菜价格预警 | 价格波动 | 郑州 | 蔬菜 | 45 | A3, B3 |

| 低风险预警 | 综合 | 全国 | 未识别 | 20 | A5 |
| Mock 故障数据 | - | - | - | - | C1-C7 |

### 通过标准
- 功能: ≥4/6 通过
- 质量: 总分 ≥ 9/12
- 降级: 7/7 全部通过
- 稳定性: 3/3 核心意思一致

## B 组：质量评估（人眼判断 vs LLM vs 规则模板）

### 评估方法

每条预警先由人眼写出"理想建议应包含的要素"，再对 LLM 输出进行 4 维度评分（每维 0-3 分）。

### LL-A1：病虫害预警（小麦赤霉病）

**预警信息**:
- 标题: 河南多地发布小麦赤霉病防治提醒
- 地区: 河南 | 产品: 小麦 | 风险类型: 病虫害
- 触发词: 赤霉病、防治、统防统治
- 评分: 78 分（较高风险）

**人眼理想建议要素**:
1. 提及小麦赤霉病这个具体病害
2. 给出具体防治措施（药剂、时机）
3. 提及河南地区

**规则模板输出**:
> 建议加强田间巡查，结合当地植保部门指导，及时开展病虫害监测和统防统治。

**LLM 输出 reason**:
> 河南多地发布小麦赤霉病防治提醒，触发较高风险评分（78分），其中关键词分（34分）和热度分（10分）较高，表明该病虫害已引起广泛关注和官方预警。当前价格变化率为0.0%，价格信号平稳，但赤霉病对小麦产量和品质的潜在威胁较大。

**LLM 输出 suggestion**:
> 建议河南地区农户立即采取统防统治措施，重点在抽穗扬花期喷施戊唑醇、咪鲜胺等高效药剂，并注意轮换用药避免抗性。同时，加强田间监测，雨后及时补防，并关注当地农业部门发布的防治技术指导。

**质量评分**:

| 维度 | 评分 | 评语 |
|------|------|------|
| 具体性 | 3/3 | 提及了赤霉病、小麦、河南、戊唑醇、抽穗扬花期等具体信息 |
| 可操作性 | 3/3 | 给出了具体的药剂名称、喷施时机（抽穗扬花期）、轮换用药建议 |
| 准确性 | 3/3 | 完全针对赤霉病防治，无张冠李戴 |
| 无幻觉 | 3/3 | 未编造任何输入中没有的信息 |
| **总分** | **12/12** | ✅ 优秀 |

**对比**: LLM 输出远优于规则模板——规则仅说"加强田间巡查"，LLM 给出了药剂名、时机、轮换策略。

## C 组：降级链路测试

| ID | 故障场景 | 模拟方式 | 预期 | 实际 | 结果 |
|----|----------|----------|------|------|------|
| LL-C1 | LLM 总开关关闭 | `AGRI_LLM_ENABLED=false` | 直接返回 | reason 保持原文 | ✅ |
| LL-C2 | API key 无效（占位符） | key=`sk-your-...` | `_init_client()` 返回 None | client=None | ✅ |
| LL-C3 | 网络超时 | monkeypatch `_call_llm` → None | reason 保留原文 | reason 未变 | ✅ |
| LL-C4 | 返回非 JSON | monkeypatch → "垃圾文本" | JSON 解析失败，原文保留 | reason 未变 | ✅ |
| LL-C5 | JSON 缺 suggestion | monkeypatch → `{"reason":"R"}` | reason 更新，suggestion 保留 | reason 更新 ✓, suggestion 不变 ✓ | ✅ |
| LL-C6 | 批量返回数量错误 | monkeypatch → 1条 (期望3条) | 仅匹配索引更新 | [0] 更新 ✓, [1][2] 不变 ✓ | ✅ |
| LL-C7 | `use_llm=False` | `generate_warnings(use_llm=False)` | 纯规则输出 | 1 条规则预警 | ✅ |

**降级测试**: 7/7 全部通过

## LLM 输出 vs 规则模板对比

| 维度 | 规则模板 | LLM 输出 |
|------|----------|----------|
| 长度 | ~30 字 | 80-120 字 |
| 产品提及 | 无 | 有（小麦/蔬菜） |
| 地区提及 | 无 | 有（河南/郑州） |
| 具体措施 | 泛泛而谈（"加强田间巡查"） | 具体步骤（"抽穗扬花期喷施戊唑醇"） |
| 评分信息利用 | 无 | 分析各子分数含义 |
| 价格信号利用 | 无 | 分析价格变化率 |
