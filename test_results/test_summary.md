## 关键发现

### ✅ 强项

1. **LLM 预警增强质量优异**：DeepSeek 生成的建议在具体性、可操作性、准确性、无幻觉四个维度上全面优于规则模板。平均评分 11.7/12。LLM 能根据具体的产品、地区、触发词生成个性化建议（如"抽穗扬花期喷施戊唑醇"对比规则模板的"加强田间巡查"）。

2. **降级链路完整可靠**：7 层降级链路全部验证通过。LLM 不可用时系统无缝回退到规则模板，不影响可用性。

3. **爬虫基本可用**：farmer.com.cn 爬取成功，离线功能全部正常。

4. **模型在短文本上优于规则**：AgriCHN 测试集上模型(61.3%)远优于规则(11.0%)，证明模型学到了文本特征。

### ❌ 弱项

1. **模型在真实新闻上表现不佳**：真实新闻上模型准确率 40% vs 规则 73%。根本原因是训练数据(AgriCHN 短句, 平均 20-50 字)与推理数据(完整新闻, 标题+正文 200-5000 字)分布不一致。

2. **模型存在"农业政策"偏好**：多条非政策新闻被高置信度误分为农业政策。

3. **moa.gov.cn 爬取失败**：CSS 选择器不匹配当前页面结构。

4. **噪声数据穿透过滤**："友情链接 网站地图"页脚链接通过了关键词过滤。

## 改进建议（按优先级）

### P0（立即）

1. **调整分类链顺序为 LLM → 规则 → 模型**：当前链路中本地模型在真实新闻上不如规则，应让规则优先。

2. **增加"无效数据"过滤**：在爬虫到分类的管道中增加质量检查，过滤掉页脚链接等非新闻内容。

### P1（短期）

3. **用真实新闻微调模型**：收集 100-200 条真实新闻并标注，在现有 BERT 基础上增量训练，提升真实场景准确率。

4. **修复 moa.gov.cn 爬虫**：检查 `moa.gov.cn/xw/` 的实际 HTML，更新 CSS 选择器。

5. **增加 OOD 检测**：对纯英文/纯符号/过短文本返回"无法分类"而非强行归类。

### P2（中期）

6. **扩大新闻源**：添加 3-5 个中文农业新闻源，提高爬取覆盖。

7. **统一批量/单条 prompt 结构**：确保两种模式下的输出质量一致。

8. **增加 LLM 分类的测试覆盖**：当前 LLM 分类路径（`llm_classifier.py`）未被系统测试。
