## 步骤 1：启动 MySQL 并初始化数据库

### 1.1 启动 MySQL 服务

以**管理员身份**打开命令提示符（Win+X → 终端(管理员)），执行：

```
net start MySQL80
```

看到 "MySQL80 服务正在启动 ... 服务已经启动成功" 即表示成功。

### 1.2 初始化数据库

双击运行项目根目录下的 `start_backend.bat`。

脚本会自动：
1. 检查并启动 MySQL 服务
2. 创建数据库 `agri_warning` 和 3 张表（news, product_prices, warnings）
3. 从 `sample_data/` 导入演示数据
4. 启动 FastAPI 后端

看到以下输出即表示成功：

```
MySQL 数据管理模块初始化完成
新闻入库: 5
价格入库: 6
预警入库: 5
```

后端启动后保持窗口打开，继续下一步。

> **如果脚本报错**：
> - "无法启动 MySQL 服务"：手动执行 `net start MySQL80`（管理员权限）
> - "数据库初始化失败"：检查 MySQL 密码是否正确（`.env` 中 `AGRI_DB_PASSWORD=password123`）

## 步骤 2：启动前端

双击运行 `start_frontend.bat`。

浏览器访问 **http://127.0.0.1:5173**

登录凭据：
- 用户名: `admin`
- 密码: `123456`

登录后应看到仪表盘页面，显示 5 条新闻、5 条预警记录。

---

## 步骤 3：联网爬取新闻

1. 在 Web 界面中，点击右上角 **"立即更新新闻"** 按钮
2. 系统将连接 `farmer.com.cn` 爬取最新农业新闻
3. 等待 5-10 秒，页面自动刷新

**预期结果**：
- 顶部出现通知：`联网更新完成：抓取 X 条，保存新闻 X 条，生成预警 X 条`
- 点击左侧 **"新闻"** 标签，查看新抓取的新闻
- 点击左侧 **"预警"** 标签，查看新生成的预警

**验证 LLM 分类**：
- 每条新闻的"分类"列显示的是 DeepSeek LLM 的分析结果
- 点击任一条新闻，查看详情中的分类和关键词

> **如果联网爬取失败**（显示 "联网更新失败"）：
> - 原因：网络问题或网站结构变化
> - 不影响后续测试，可以直接跳到步骤 4 使用手动导入数据

## 步骤 4：手动导入测试数据

### 4.1 导入准备好的测试新闻

1. 在 Web 界面中，点击左侧 **"导入"** 标签
2. 点击 **"选择文件"**，选择 `test_data/import_news_test.csv`
3. 点击 **"解析文件"**

**预期结果**：
- 下方显示 JSON 结果：`{"message": "file parsed", "rows": 6, "sample": [...]}`
- 6 条测试新闻被解析

### 4.2 重建演示数据（包含导入的新闻）

如果步骤 3 爬取成功且步骤 4.1 导入成功，可以跳过此步。

如需重建：在浏览器地址栏访问（或使用 API 测试工具）：
```
http://127.0.0.1:8000/api/pipeline/rebuild
```
（这是一个 POST 请求，可在 `/docs` 页面中点击 Try it out 执行）

## 步骤 5：验证 LLM 分类结果

### 5.1 查看新闻分类

1. 在 Web 界面中点击 **"新闻"** 标签
2. 查看每条新闻的 **"分类"** 列

**预期**：6 条测试新闻的分类应与上表中的类别一致

### 5.2 验证 LLM 工作状态（通过 API 文档）

在浏览器打开 **http://127.0.0.1:8000/docs**

找到 `GET /api/news` 接口：
1. 点击展开
2. 点击 **"Try it out"**
3. 点击 **"Execute"**

查看返回的 JSON 数据，确认每条新闻包含：
- `category`: 分类结果
- `summary`: 摘要
- `keywords`: 关键词

---

## 步骤 6：验证 LLM 预警建议

### 6.1 查看预警列表

1. 在 Web 界面中点击 **"预警"** 标签
2. 查看预警列表

**关键验证点**：

对于"暴雨橙色预警"类预警，LLM 建议应包含：
- ✅ 具体提及"河南"地区
- ✅ 具体提及"清沟排水"、"大棚加固"等措施
- ✅ 风险评分分解条（关键词分、价格分、热度分等已显示）
- ✅ 建议文本长度 > 50 字（规则模板仅 ~30 字）

对比：如果是规则模板，只会显示固定文本如"建议关注当地气象和农业农村部门通知..."

### 6.2 验证离线模式（可选）

如果想验证 LLM 不可用时的降级行为：

1. 编辑 `data_management/.env`，将 `AGRI_LLM_ENABLED=true` 改为 `false`
2. 重新启动后端
3. 在 `/docs` 页面执行 `POST /api/pipeline/rebuild`
4. 查看预警的 `suggestion` 字段——此时应显示较短的规则模板文本
5. 测试完成后改回 `true` 并重启

## 常见问题

| 问题 | 解决方法 |
|------|----------|
| MySQL 无法启动 | 以管理员运行 `net start MySQL80` |
| 后端启动报错 | 检查 `.env` 中密码是否为 `password123` |
| 前端无法访问 | 确认后端已启动，检查防火墙 |
| LLM 不工作 | 检查 `secrets.env` 中 API key 是否有效，网络能否访问 `api.deepseek.com` |
| 爬虫无数据 | farmer.com.cn 可能改版，手动导入测试数据即可验证其他功能 |

---

## 关闭测试环境

1. 关闭两个命令行窗口（停止后端和前端）
2. MySQL 服务保持运行即可（下次启动后端时无需重新初始化）
